# PROJECT.md

This document is the main working guide for TinCapOS.

## Purpose

TinCapOS is being prepared as a clean foundation for future product development.

## Current Scope

- establish repository structure
- keep the codebase ready for Git-based collaboration
- avoid business logic at this stage
- avoid database schemas at this stage
- avoid feature implementation at this stage

## Repository Structure

- `backend/`
- `frontend/`
- `docs/`
- `database/`
- `storage/`
- `tests/`
- `scripts/`

## Working Rules

- make changes through Git-tracked files only
- keep the foundation clean and minimal
- document architectural decisions in `docs/` when needed
